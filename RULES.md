# 数学物理方法知识可视化与AI互动教学 - 项目规则

## 一、项目终极目标

### 最终产品形态
一个基于Streamlit的Web应用，包含两大核心模块：

**模块A：知识可视化**
- 将数学物理方法中的核心概念转化为交互式图表
- 用户通过滑块、按钮等控件调节参数，实时观察图形变化
- 示例：复变函数映射动画、偏微分方程解演化、特殊函数图鉴

**模块B：AI互动教学**
- 用户输入某个章节（如"分离变量法"），内嵌AI将化身一对一家教
- AI的教学流程必须包含：引导提问 → 讲解概念 → 出题 → 批改 → 纠错 → 总结
- 不准直接给出最终答案，需用苏格拉底式提问逐步启发
- 根据学生回答的对错，动态调整后续题目难度

## 二、角色分工

### 学生（项目大脑）
- 负责物理逻辑、教学流程设计
- 理解数学物理方法的公式和概念

### 助手（项目双手）
- 负责所有代码实现、Debug、环境配置
- 编写详细中文注释

## 三、技术约定

1. **语言**：Python 3.10+
2. **界面框架**：Streamlit（优先），必要时可引入Plotly/Dash
3. **绘图库**：Matplotlib（基础）、Plotly（交互）
4. **注释规范**：每行代码必须附带详细中文注释，解释"物理含义"和"编程目的"
5. **环境管理**：每个独立功能需生成对应的 requirements.txt
6. **回答风格**：默认用"面向物理专业的Python教学"风格解释

## 七、已知问题与解决思路

### ✅ 已解决问题
**复变函数模块的数组形状错误**
- 错误：`ValueError: all input arrays must have the same shape`
- 位置：`modules/complex_functions.py` 的 `calculate_color_values` 函数
- 修复方法：重写颜色计算函数，使用numpy数组操作而非标量广播
- 状态：已完全修复并测试通过

### 问题1：Streamlit缓存导致代码不更新
**症状**：修改代码后错误仍然存在
**尝试过的解决方法**：
- 删除 `__pycache__` 目录
- 删除 `.pyc` 文件
- 设置 `PYTHONDONTWRITEBYTECODE=1`
- 重启Streamlit服务器
- 完全重启所有Python进程

**未尝试的解决思路**：
- 检查是否有全局Python缓存目录
- 考虑使用 `pip install -e .` 重新安装项目
- 检查Streamlit是否有额外的缓存机制

### 问题2：命令行导入成功但Streamlit导入失败
**可能原因**：
- Streamlit使用不同的Python环境
- Streamlit有模块预加载机制
- 存在命名冲突
